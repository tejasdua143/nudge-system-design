Create Presentation from Content

🚧 Beta API Notice

The Create from Content API is currently in beta. Functionality, rate limits, and pricing are subject to change.

This endpoint allows you to create presentations from structured text content. Pass your own content directly in the request body and let AI transform it into a professional presentation.

POST /api/v1/document/content

Creates a presentation from structured content provided directly in the request body.

Authorization

Include your API key in the Authorization header as a Bearer token.

Request Parameters

Field Type Required Description
content string Yes Raw text content to convert.
topic string No Topic or instruction text.
slideCount number No Number of slides (1-50). If omitted, forces preserve mode.
language string No Language for generation.
domain string No Context domain.
exportType string Yes File type (.ppt, .pptx, .pdf, .image, .render, .share).
instruction string No Mode: enhance, summarize, instruction.
targetAudience string No Target audience.
callback_url string No HTTPS URL for async callback.
immediatePollUrl boolean No If true, returns poll URL immediately.

Callback URL Flow

If you provide a valid callback_url, the API responds immediately with a job ID:

{
    "job_id": "c2fe7146-2983-50e2-c884-45c031f85315",
    "status": "queued"
}

When processing is complete, the result will be POSTed to your callback URL. If you do not provide a callback_url, the result is returned directly in the response.

Timeout & Polling (no callback_url)

If processing exceeds 5 minutes, you will receive:

{
  "status": 1,
  "pollUrl": "https://developers.presentations.ai/api/v1/polljob/<job_id>"
}

Poll with GET https://developers.presentations.ai/api/v1/polljob/<job_id> using the same Bearer token until complete.

Error Handling

{ "status": 1, "error": "Please provide content text" }

Solution: Ensure the content field is not empty.

{ "status": 1, "error": "Content exceeds maximum length" }

Solution: Reduce your content length or split into multiple requests.

{ "status": 1, "error": "Invalid callback_url. Must be a valid HTTPS URL" }

Solution: Provide a valid HTTPS URL for the callback.

Usage Notes

  • Content Quality: Well-structured content with clear sections produces better slide layouts.
  • Topic Guidance: Use the topic field to give additional instructions to the AI about how to structure the presentation.
  • Tone & Audience: Specify tone and targetAudience for more tailored output.
  • Callback URL: Use async callbacks for large content to avoid timeouts.

For more details about error handling, see our Error Handling Guide.
