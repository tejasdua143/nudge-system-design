⚡ Quick Start Guide

This guide walks you through creating your first presentation using the Presentations.AI API.

🎯 What You'll Learn
  • Account setup and API key generation
  • Your first API call
  • Understanding responses and next steps
  • Testing with different programming languages

🚀 Step 1: Create Your Account

  1. Visit developers.presentations.ai/docs/login
  2. Sign up using:
    • LinkedIn, Google, or Microsoft (instant access)
    • Email (requires email confirmation)
  3. Complete the registration process
✅ Free Credits

New accounts receive 200 free credits to get started!

🔑 Step 2: Get Your API Key

  1. Navigate to Settings → API / Register for API
  2. Click "Register for API" button
  3. Click "Reveal" to display your API key
  4. Copy and store your API key securely
🔒 Security Note

Never commit API keys to version control or share them publicly. Store them as environment variables or in secure configuration.

🎯 Step 3: Make Your First API Call

Let's create a simple presentation about "AI in Healthcare":

Using cURL:

curl -X POST https://developers.presentations.ai/api/v1/topic/document \
            -H "Authorization: Bearer YOUR_API_KEY_HERE" \
            -H "Content-Type: application/json" \
            -d '{
                "topic": "AI in Healthcare: Transforming Patient Care",
                "slideCount": 6,
                "language": "en", 
                "domain": "presentations.ai",
                "exportType": "ppt"
            }'

Using JavaScript (Node.js):

const axios = require('axios');

                const createPresentation = async () => {
                try {
                    const response = await axios.post('https://developers.presentations.ai/api/v1/topic/document', {
                    topic: "AI in Healthcare: Transforming Patient Care",
                    slideCount: 6,
                    language: "en",
                    domain: "healthcare", 
                    exportType: "ppt"
                    }, {
                    headers: {
                        'Authorization': 'Bearer YOUR_API_KEY_HERE',
                        'Content-Type': 'application/json'
                    }
                    });
                    
                    console.log('Success:', response.data);
                    return response.data;
                } catch (error) {
                    console.error('Error:', error.response.data);
                }
                };

                createPresentation();

Using Python:

import requests
                import json

                def create_presentation():
                url = "https://developers.presentations.ai/api/v1/topic/document"

                headers = {
                    "Authorization": "Bearer YOUR_API_KEY_HERE",
                    "Content-Type": "application/json"
                }

                data = {
                    "topic": "AI in Healthcare: Transforming Patient Care",
                    "slideCount": 6,
                    "language": "en",
                    "domain": "presentations.ai",
                    "exportType": "ppt"
                }

                try:
                    response = requests.post(url, headers=headers, json=data)
                    response.raise_for_status()
                    
                    result = response.json()
                    print("Success:", json.dumps(result, indent=2))
                    return result
                    
                except requests.exceptions.RequestException as e:
                    print(f"Error: {e}")

                create_presentation()

🎉 Step 4: Understanding the Response

A successful response looks like this:

{
  "status": 0,
  "docid": 199943,
  "docurl": "https://cdn.presentations.ai/exports/199943/1755583984885/Digital%20Marketing%20for%20Small%20Businesses.pptx",
  "animated_url": "https://devcdn.presentations.ai/exports/199943/1755583984566/Digital%20Marketing%20for%20Small%20Businesses_animated.pptx"
}

Response Fields:

  • status: 0 = success, 1 = error
  • message: Human-readable status message
  • docurl: Direct link to view/edit your presentation
  • insertid: Unique presentation ID for future operations
Success

Open the docurl to view your presentation.

❌ Common Errors & Solutions

Authentication Error (401):

{
  "status": 1,
  "error": "Unauthorized access"
}

Solution: Check your API key and Authorization header format.

Insufficient Credits (402):

{
  "status": 1,
  "error": "Insufficient credits"
}

Solution: Log in at href="https://developers.presentations.ai" class="text-blue-600 hover:text-blue-800 underline">developers.presentations.ai to check your balance and purchase more credits.

Invalid Topic (400):

{
  "status": 1,
  "error": "Topic cannot be empty"
}

Solution: Ensure your topic is descriptive and not empty.
